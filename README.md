# Telegram Support Bot

> Этот проект представляет собой Telegram-бота для организации простейшей, но удобной системы поддержки.
> Бот принимает сообщения пользователей (текст, фото, видео, аудио, документы, голосовые) и пересылает их в чат поддержки.
> Операторы отвечают на сообщения прямо в общем чате, используя функцию Reply, а бот автоматически перенаправляет ответ пользователю.

*Это мой первый публичный репозиторий, поэтому не ругайте строго ✨*

## 🚀 Возможности

* **🔄 Двусторонняя связь:** Пересылка сообщений от пользователей в чат поддержки и обратно со всеми медиа-вложениями.
* **🔘 Интерактивное управление:** Закрытие/открытие тикетов и блокировка пользователей осуществляются в один клик через динамические инлайн-кнопки прямо под сообщениями.
* **🕒 Режим работы (Автоответчик):** Возможность задать рабочие часы поддержки. Если пользователь напишет в нерабочее время, бот автоматически предупредит его о задержке с ответом.
* **📁 Гибкая система топиков:** - **Режим "Отдельный топик для каждого"** — автоматическое создание персонального топика для каждого пользователя с визуальным отображением статуса (🟢 открыт / 🔴 закрыт).
* **Режим "Общий топик"** — все тикеты собираются в одном топике (классический режим).


* **⚙️ Админ-панель:** Администраторы могут менять тексты приветствия, справки, настраивать автоответчик и переключать режим работы топиков прямо через бота в Telegram.
* **💾 Модульная архитектура:** Код разделен на логические блоки (`bot.py`, `admin.py`, `database.py`) и использует локальную базу данных SQLite.

## 📋 Требования

* Python 3.8+
* Токен Telegram бота
* ID чата (группы), где будут находиться операторы поддержки
* **Для режима "Общий топик"**: ID топика (`SUPPORT_TOPIC_ID`)
* **Для режима "Отдельный топик для каждого"**: Чат должен быть супергруппой с включенными темами, бот должен быть администратором с правами `Управление темами`
* ID администраторов (для доступа к настройкам текстов и режимов работы)

## 🛠 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/gniloeapple/telegram-support-bot.git
cd telegram-support-bot

cp .env.example .env
nano .env

```

**Пример заполнения `.env`:**

```ini
TELEGRAM_TOKEN=123456789:AAH...             # Токен от BotFather
SUPPORT_CHAT_ID=-1001234567890              # ID группы поддержки (начинается с -100)
SUPPORT_TOPIC_ID=12345                      # Опционально: ID топика (для режима "Общий топик")
ADMINS=123456789,987654321                  # ID администраторов через запятую (без пробелов)

```

**Где взять данные:**

* **TELEGRAM_TOKEN**: Напишите [@BotFather](https://t.me/BotFather), создайте бота и скопируйте выданный токен.
* **SUPPORT_CHAT_ID**:
1. Создайте группу в Telegram, добавьте туда бота и назначьте его администратором.
2. Перешлите любое сообщение из группы боту `@userinfobot`, чтобы узнать ID группы. Обязательно убедитесь, что ID начинается с `-100`.


* **SUPPORT_TOPIC_ID**:
* Нужен только для режима "Общий топик". Включите темы в группе и перешлите сообщение из нужной темы в `@userinfobot`, чтобы узнать `message_thread_id`.


* **ADMINS**: Ваш цифровой ID в Telegram.

### 2. Установка зависимостей

Рекомендуется использовать виртуальное окружение:

```bash
python -m venv venv
# Активация для Windows:
venv\Scripts\activate
# Активация для Linux/macOS:
source venv/bin/activate

pip install -r requirements.txt

```

### 3. Запуск бота

```bash
python bot.py

```

## 📖 Как использовать

### Для пользователей

1. Запускают бота командой `/start`.
2. Пишут сообщение (поддерживаются текст, фото, видео, документы, голосовые).
3. Бот пересылает обращение в поддержку. Если сейчас нерабочее время (и оно настроено), пользователь получит соответствующее предупреждение.

### Для операторов поддержки

1. **Ответ на сообщение:** Чтобы ответить пользователю, сделайте **Reply (Ответить)** на пересланное сообщение в группе. Бот доставит ответ в личные сообщения пользователя.
2. **Управление тикетом:** Под каждым сообщением есть инлайн-кнопки **«🔒 Закрыть тикет» / «🔓 Открыть тикет»** и **«❌ Заблокировать» / «✅ Разблокировать»**. При нажатии статус меняется динамически.
3. Дополнительные команды в чате поддержки:
* `/open_tickets` — выводит список всех открытых обращений с деталями.
* `/ticket` (по реплаю)
  Показывает краткую информацию: ID пользователя, статус тикета, **статус блокировки (Да/Нет)**, дата создания и обновления.



## 🤖 Меню администратора

Для вызова панели настроек отправьте боту в личные сообщения команду `/admin` (доступно только пользователям из списка `ADMINS` в `.env`).

В меню доступно:

* **✏️ Изменить приветствие** — задать текст для `/start`.
* **📝 Изменить информацию** — задать текст для `/help`.
* **🕒 Режим работы** — включение/выключение автоответчика в нерабочие часы, установка расписания (например, `10:00-18:00`) и текста автоответа.
* **🔄 Режим: [Текущий режим]** — переключение логики топиков:
* 📁 **Отдельный топик для каждого** — каждый новый пользователь получает свой топик. При закрытии/открытии тикета индикатор в названии топика меняется (🔴/🟢).
* 📂 **Общий топик** — все тикеты сыплются в одну кучу.


* **❌ Закрыть** — удаляет сообщение с меню админ-панели, чтобы не засорять чат.
